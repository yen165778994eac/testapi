# Анализ сетевых запросов через DevTools

## Цель
Показать, как я использую DevTools для перехвата и анализа HTTP-запросов при тестировании веб-приложений.

## Что было сделано
1. Открыта страница `https://oort.qahacking.ru/novosti-stancii/`.
2. В DevTools (F12) → вкладка **Network** перехвачены POST-запросы:
   - Форма поиска груза (`form_id=710`)
   - Форма заявки на членство в клубе (`form_id=745`)
3. Проанализированы заголовки, тело, куки и ответы сервера.
4. Проведены позитивные и негативные тесты с изменением параметров (удаление полей, невалидные данные).
5. Результаты задокументированы в Postman.

---

## Результаты анализа

### 1. Форма поиска груза

| Что проверяли | Результат |
| :--- | :--- |
| URL запроса | `https://oort.qahacking.ru/wp-admin/admin-ajax.php` |
| Метод | `POST` |
| Тип контента | `multipart/form-data` |
| Ключевые поля | `Поиск`, `action`, `fusion_form_nonce`, `form_id=710` |
| Успешный ответ | `200 OK` с телом `{"status":"success","info":"form_submitted"}` |
| Ошибка (пустое тело) | `400 Bad Request` |

### 2. Форма заявки на членство

| Что проверяли | Результат |
| :--- | :--- |
| URL запроса | `https://oort.qahacking.ru/wp-admin/admin-ajax.php` |
| Метод | `POST` |
| Тип контента | `multipart/form-data` |
| Ключевые поля | `action`, `fusion_form_nonce`, `form_id=745`, `phone_number` и др. |
| Успешный ответ | `200 OK` с `{"status":"success","info":"form_submitted"}` |
| Ошибка (пустое тело) | `400 Bad Request` |

## Проведённые тесты

| Сценарий | Что меняли | Код ответа | Тело ответа | Результат |
| :--- | :--- | :--- | :--- | :--- |
| **Поиск груза (позитивный)** | `Поиск=12` | 200 OK | `{"status":"success"...}` | ✅ PASS |
| **Поиск груза (пустое тело)** | Удалили все поля | 400 Bad Request | (пусто) | ✅ PASS (ожидаемая ошибка) |
| **Заявка (позитивный)** | Все поля заполнены | 200 OK | `{"status":"success"...}` | ✅ PASS |
| **Заявка (пустое тело)** | Удалили все поля (none) | 400 Bad Request | (пусто) | ✅ PASS |
| **Заявка (пустой телефон)** ⚠️ | Удалили `phone_number` | 200 OK | `{"status":"success"...}` | ❌ **FAIL** (ожидалась ошибка) |
| **Заявка (текст в телефоне)** ⚠️ | Вставили "Text" в `phone_number` | 200 OK | `{"status":"success"...}` | ❌ **FAIL** (ожидалась ошибка) |

> **Найденные баги:** сервер принимает заявки с пустым и невалидным телефоном. Подробности в [`bugs-found.md`](bugs-found.md).
